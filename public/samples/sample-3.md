# 개발팀 진단보고서 샘플

## 요약
PR 리뷰 시 코드 컨벤션·간단 버그를 사람이 매번 지적. 시니어 개발자가 주당 6~10시간 리뷰에 소비.

## 추천 AI 도구
- **GitHub Copilot Code Review**: PR 자동 1차 리뷰
- **CodeRabbit**: 변경 요약 + 잠재 버그 감지
- **Claude API**: 도메인 규칙 기반 커스텀 lint

## 시간 절감 예측
주 6~10시간 → 주 2시간 (약 70% 절감, 시니어는 도메인 판단에만 집중)

## 자동화 흐름
1. PR 생성 시 GitHub Action 트리거
2. CodeRabbit이 diff 분석·요약·이슈 코멘트
3. Claude API가 사내 도메인 규칙(예: 결제 모듈 보안 가드) 검증
4. 사람 리뷰어는 비즈니스 로직·아키텍처에만 집중

## 코치 코멘트
도구 신뢰 구축이 핵심. 첫 1~2개월은 사람 리뷰와 병행하며 false positive를 정리하고, 팀 컨벤션을 학습 자료로 추가.
